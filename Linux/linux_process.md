# ps(process status)
```
ps：显示与当前用户和控制台相关的进程
参数：
a 列出带有终端的所有用户的进程
x 列出当前用户的所有进程，包括没有终端的进程
u 面向用户友好的显示风格
-e 列出所有进程
-u 列出某个用户关联的所有进程阿萨德
-f 显示完整格式的进程列表
-为标准unix风格，无-为BSD风格
ps aux：查看系统中所有进程
ps -ef：可以查看父子进程之间的关系
Ps：查看CPU和内存占用率用ps aux，查看父进程ID用ps -ef
```

# ps aux显示说明
```
USER：进程由哪个用户产生
PID： 进程ID号
%CPU：该进程占用CPU百分比
%MEM：该进程占用物理内存百分比
VSZ： 该进程占用虚拟内存的大小
RSS： 该进程占用实际物理内存的大小
TTY： 该进程在哪个终端中运行，对于CentOS来说，tty1是图形化终端，tty2-6是字符界面终端，pts/0-255代表虚拟终端
STAT：进程状态，R-运行，S-睡眠，T-暂停，Z-僵尸，s-包含子进程，l-多线程，+-前台显示
START：该进程的启动时间
TIME： 该进程占用CPU的运算时间
COMMAND：产生此进程的命令
```

# ps -ef显示说明
```
......
PPID：父进程ID
C：   CPU用于计算执行优先级的因子，数值越大说明是CPU密集型运算，执行优先级会降低，越小是IO密集型运算，优先级会提高
```

# kill
```
kill 选项 进程号：通过进程号杀死进程
killall 进程名称：通过进程名称杀死进程
-9 强迫进程立即停止
kill -l：查看信号值含义
```

# top
```
top 选项：实时监控系统进程状态
-d：指定top每隔几秒更新，默认3秒
-i：不显示任何闲置或者僵死进程
-p：指定进程PID来仅监控该进程
显示信息：
up：开机到目前持续的时间
load average：系统平均负载（1分钟，5分钟，15分钟）
%CPU（s）：us（没有经过用户更改优先级的进程）sy（系统进程）ni（经过nice调整优先级的进程）id（空闲时间）wa（等待IO时间）hi（硬件中断）si（软件中断）st（虚拟化时间占用）
VIRT：虚拟内存占用大小
RES：物理内存占用大小
SHR：共享内存占用大小
S：State状态
操作：
M：按内存占用排序
P：按CPU占比排序
N：按进程号从大到小排序
u：指定查看某个用户的进程
k：给定PID杀死进程
q：退出
```

# netstat
```
netstat -anp | grep 进程号：查看该进程网络信息
netstat -nlp | grep 进程号：查看网络端口号占用情况
-a 显示所有正在监听和未监听的套接字
-n 拒绝显示别名，能用数字显示就用数字
-l 仅列出在监听的服务状态
-p 表示显示哪个进程在调用
```
